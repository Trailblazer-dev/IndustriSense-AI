What is IndustriSense AI?

IndustriSense AI is an advanced predictive maintenance system powered by machine learning. It analyzes machine operational data to predict failures and estimate remaining useful life, enabling proactive maintenance scheduling and reducing unexpected downtime.

Key Features

⚡ Real-Time Predictions

Instant failure risk assessment based on current machine parameters

🔮 RUL Estimation

Accurate remaining useful life predictions for maintenance planning

📊 Advanced Analytics

Feature importance analysis and model performance metrics

🎯 Risk Classification

Automatic categorization into Critical, Warning, and Normal states

🔍 Explainability

Understand why predictions are made with detailed feature analysis

💻 User-Friendly Dashboard

Intuitive interface designed for maintenance operators and engineers

System Architecture

The system consists of two complementary ML models:

Failure Classification Model

Type: XGBoost Binary Classifier

Purpose: Predicts probability of equipment failure (0-100%)

Features: 10 engineered parameters including temperature differentials and operational interactions

Output: Risk classification (Critical/Warning/Normal) and confidence probability

RUL Prognosis Model

Type: XGBoost Regressor

Purpose: Estimates tool wear and remaining useful life

Features: 9 engineered parameters for wear prediction

Output: Tool wear in minutes and RUL countdown

How It Works

  1. Data Collection

    Real-time machine operational parameters (temperature, speed, torque, etc.)

  2. Feature Engineering

    Transform raw parameters into 10 meaningful features including interaction terms

  3. Model Prediction

    Both classifier and regressor generate predictions simultaneously

  4. Risk Assessment

    Classify machine status and estimate time until maintenance needed

  5. Action & Decision

    Operators receive alerts and recommendations for maintenance scheduling

Critical Limitations

⚠️ Important Considerations

  • Class Imbalance: Model exhibits extreme imbalance (96.6% normal, 3.4% failure). Predictions should be interpreted as relative risk rankings rather than absolute probabilities.
  • Snapshot-Based: Predictions are based on instantaneous machine state, not temporal degradation patterns.
  • Static Training: Model was trained on historical data; accuracy may degrade with distribution shift.
  • Feature-Limited: Predictions depend on availability of required operational parameters.
  • No Real-Time Monitoring: Current system provides batch predictions, not continuous streaming analysis.
